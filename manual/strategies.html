Strategies

Strategies are the building blocks of a pipeline. Each implements ILootStrategy and receives the working set (mutable roll state) and context (read-only caller data).

public interface ILootStrategy<T> {
    void Process(LootWorkingSet<T> workingSet, Context context);
}

Strategies are executed in the order they are added to the pipeline.

Strategy Ordering

Important

Order matters. A typical pipeline flows like this:

[ModifyWeight]  →  [Selection]  →  [Pity]  →  [Filter]  →  [PostProcess]
Phase Strategies
Pre-roll weight scaling ModifyWeightStrategy
Selection WeightedRandomStrategy, DropChanceStrategy
Guarantee / fallback GuaranteedDropStrategy, PityStrategy, ItemPityStrategy
Filtering FilterStrategy, FilterByContextStrategy
Post-processing ModifyQuantityStrategy, LimitResultsStrategy, ConsolidateResultsStrategy, ExpandResultsStrategy, BonusRollStrategy, UniqueDropStrategy, NestedTableStrategy

Selection Strategies

WeightedRandomStrategy

Picks rollCount items from the table using cumulative weighted random selection.

// Roll 3 times
new WeightedRandomStrategy<Item>(rollCount: 3);

// Roll once, no duplicates
new WeightedRandomStrategy<Item>(rollCount: 1, allowDuplicates: false);
Note

The same item can appear multiple times in the results if allowDuplicates is true (default).

DropChanceStrategy

Rolls every entry independently. Each entry's Weight is treated as a probability, not a relative weight.

// Weights are 0�1 (0.05 = 5% chance)
new DropChanceStrategy<Item>();

// Weights are 0�100 (5.0 = 5% chance)
new DropChanceStrategy<Item>(weightAsPercent: true);
Tip

This is useful for tables where each entry should be evaluated separately rather than competing with each other.

Guarantee / Fallback Strategies

GuaranteedDropStrategy

Only activates when the result list is empty. Performs one weighted random selection from the table.

pipeline.AddStrategy(new GuaranteedDropStrategy<Item>());
Tip

Place this after your selection strategies so it only fires when they produce nothing.

Pity Strategies

See Pity Systems for full details.

PityStrategy

Hard pity: after threshold consecutive rolls that produce no result, the next roll always succeeds. Resets on success. Tracks by table ID or group key.

ItemPityStrategy

Hard pity for a specific item. Guarantees the item drops after it has failed to drop N times.

Pre-roll Modification

ModifyWeightStrategy

Important

Scale entry weights before selection. Must be placed before any selection strategy.

// Multiply all weights by a fixed factor
pipeline.AddStrategy(ModifyWeightStrategy<Item>.Multiplier(2f));

// Read multiplier from context
pipeline.AddStrategy(ModifyWeightStrategy<Item>.MultiplierFromContext(luckKey));

// Scale based on context value range
pipeline.AddStrategy(ModifyWeightStrategy<Item>.ScaleByContextRange(
    contextKey,
    range: new(1, 60),
    multiplier: 2.5f
));

// Custom per-entry scaling
pipeline.AddStrategy(new ModifyWeightStrategy<Item>((entry, context) => {
    float @base = entry.Weight;
    return entry.Entry.Item is RareItem ? @base * 2f : @base;
}));
Important

Recalculates cumulative weights and TotalWeight after modification so selection strategies see the updated distribution.

Filtering Strategies

FilterStrategy

Removes results where the predicate returns false.

// Remove items below quality threshold
pipeline.AddStrategy(new FilterStrategy<Item>(result => result.Item.Quality >= Rarity.Rare));

FilterByContextStrategy

Like FilterStrategy but the predicate also receives the context:

pipeline.AddStrategy(new FilterByContextStrategy<Item>((result, context) => {
    if (!context.TryGet(playerLevelKey, out int level))
        return true; // keep if no level context
    return result.Item.MinLevel <= level;
}));

Quantity Modification

ModifyQuantityStrategy

Adjusts the quantity of each result after selection.

// Multiply by a fixed factor (rounds down, min 1)
pipeline.AddStrategy(ModifyQuantityStrategy<Item>.Multiplier(2));

// Multiply by a context value
pipeline.AddStrategy(ModifyQuantityStrategy<Item>.MultiplierFromContext(quantityBonusKey));

// Custom logic
pipeline.AddStrategy(new ModifyQuantityStrategy<Item>((qty, context) =>
    context.GetOrDefault(isBossKey) ? qty * 3 : qty));

Result Count Strategies

LimitResultsStrategy

Caps the result list at maxResults. Excess items are removed from the end.

pipeline.AddStrategy(new LimitResultsStrategy<Item>(maxResults: 5));

LimitResultsFromContextStrategy

Reads the limit from a context key:

pipeline.AddStrategy(new LimitResultsFromContextStrategy<Item>(maxDropsKey));

Post-processing Strategies

ConsolidateResultsStrategy

Merges duplicate items into a single result by summing their quantities.

// Use default equality (Equals/GetHashCode)
pipeline.AddStrategy(new ConsolidateResultsStrategy<Item>());

// Custom comparer (e.g. compare by item ID only, ignore quality)
pipeline.AddStrategy(new ConsolidateResultsStrategy<Item>(new ItemIdComparer()));

BonusRollStrategy

Reads a 0–1 chance from the context and adds one extra weighted random roll if the chance succeeds.

// Keys.Luck = 1.0 → 100% bonus roll; 0.0 → no bonus roll
.AddStrategy(new BonusRollStrategy<Item>(Keys.Luck))

UniqueDropStrategy

Stateful. Tracks items that have already dropped (per instance) and removes duplicates from future results.

var uniqueDrops = new UniqueDropStrategy<Item>();

// Later — reset between sessions or dungeon runs
uniqueDrops.ResetAll();   // or uniqueDrops.Reset(specificItem)

// Check / manually mark
uniqueDrops.HasDropped(sword);      // bool
uniqueDrops.MarkAsDropped(sword);   // manual override

NestedTableStrategy

When an item acts as a "table selector", resolve it against a sub-table and replace the item with the results of that sub-table.

// Provide a resolver that maps an item to its sub-table
.AddStrategy(new NestedTableStrategy<TableKey>(
    tableResolver: key => tableRegistry[key],
    nestedPipeline: innerPipeline // optional; defaults to WeightedRandom(1)
))
Note

Items that don't resolve to a table (resolver returns null) are kept as-is.

ExpandResultsStrategy

Expands individual results into multiple items via a delegate. Useful for "loot set" items that contain a collection of actual drops.

.AddStrategy(new ExpandResultsStrategy<Item>(
    expander: item => lootSetDatabase[item], // returns IEnumerable<Item>? or null to skip
    quantityResolver: item => new IntRange(1, 3) // optional per-expanded-item quantity
))
Note

Items where the expander returns null are kept unchanged.

Writing a Custom Strategy

Implement ILootStrategy<T>. You have full access to the working set's results, weighted entries, blackboard, random, and source table:

public class ElementalFilterStrategy<T> : ILootStrategy<T> where T : IElemental {
    readonly Element _allowedElement;

    public ElementalFilterStrategy(Element allowedElement)
        => _allowedElement = allowedElement;

    public void Process(LootWorkingSet<T> workingSet, Context context) {
        for (var i = workingSet.Results.Count - 1; i >= 0; i--) {
            var item = workingSet.Results[i].Item;
            if (item != null && item.Element != _allowedElement)
                workingSet.Results.RemoveAt(i);
        }
    }
}

Accessing the random source

float roll = workingSet.Random.Value; // 0–1 exclusive
int idx  = workingSet.Random.Range(0, 5); // exclusive upper bound

Inter-strategy communication via Blackboard

public static class MyKeys {
    public static readonly Key<int> RollCount = new("RollCount");
}

// Strategy A — store data
workingSet.Blackboard.Set(MyKeys.RollCount, rollCount);

// Strategy B — read data
if (workingSet.Blackboard.TryGet(MyKeys.RollCount, out int n))
    // use n

The blackboard is a Context cleared between rolls. Use Key<T> for type-safe access.

Pipeline introspection

IReadOnlyList<ILootStrategy<T>> steps = pipeline.Strategies;

// Insert before an existing strategy
int idx = pipeline.Strategies.IndexOf(existingStrategy);
pipeline.InsertStrategy(idx, myNewStrategy);

// Remove a strategy
pipeline.RemoveStrategy(existingStrategy);
