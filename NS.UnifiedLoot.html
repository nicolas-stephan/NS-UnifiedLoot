Classes

BonusRollStrategy<T>

Adds bonus rolls based on context values (like luck or magic find).

CompositeTableBuilder<T>

Builder for creating a flattened, immutable loot table from multiple sub-tables. Replaces the runtime CompositeTable.

ConsolidateResultsStrategy<T>

Consolidates duplicate items by combining their quantities. Useful after strategies that might produce multiple rolls of the same item.

Context

A flexible context container.

DropChanceStrategy<T>

Rolls each entry independently against its weight as a drop chance (0-1 or 0-100). Unlike weighted random, entries don't compete - each is rolled separately.

ExpandResultsStrategy<T>

Replaces individual results with an expanded collection — useful for "drop sets" where one logical item resolves into a group of concrete items.

For each result, the expander delegate is called. If it returns a non-null collection, the original result is removed and replaced with one LootResult<T> per expanded item (quantity defaults to 1 unless an optional quantityResolver is provided). If the delegate returns null, the result passes through unchanged.

FilterByContextStrategy<T>

Filters results based on a predicate that has access to context.

FilterStrategy<T>

Filters results based on a simple predicate.

GuaranteedDropStrategy<T>

Ensures at least one item drops if the current results are empty. Useful as a fallback at the end of a pipeline.

ItemPityStrategy<T>

A strategy that tracks pity for specific items rather than the whole table. If a tracked item hasn't dropped for a certain number of rolls, it is guaranteed.

Key
Key<T>

A type-safe key for storing and retrieving values from a Context.

LimitResultsFromContextStrategy<T>

Limits results based on a context value.

LimitResultsStrategy<T>

Limits the number of results to a maximum count.

LootEntry
LootEntryDataBase

Non-generic base for LootTableAsset<TItem> entries. Targeted by the property drawer so that weight, quantity and note render with probability hints in the Inspector.

LootPipeline<T>

A composable pipeline of strategies that process loot rolls. The pipeline defines HOW loot is processed; the table defines WHAT can drop.

LootPreviewEntry<T>

A potential outcome in a loot table preview.

LootPreviewer

Utility for calculating the "dry run" state of a loot table processed by a pipeline. Useful for showing users what a loot table looks like after modifications.

LootTableAssetBase

Non-generic base for editor tooling. Not intended for direct use.

Entry data stored in the asset.

LootTableIdGenerator

Helper for generating unique table IDs.

LootTablePreview<T>

Represents the state of a loot table after being processed by a pipeline, without actually performing any random rolls.

LootTable<T>

A simple code-defined loot table.

LootWorkingSet<T>

The working state passed through the loot pipeline. Strategies read from and write to this set during processing.

ModifyQuantityStrategy<T>

Modifies the quantity of all results.

ModifyWeightStrategy<T>

Scales the weights of weighted entries before selection strategies run. Must be placed before WeightedRandomStrategy<T> in the pipeline.

NestedTableStrategy<T>

Allows rolling on additional tables from the results. Useful for nested loot pools (e.g., a "weapon" entry that rolls on a weapon-specific table). For each matched result, the nested pipeline is executed once per quantity unit.

PityStrategy<T>

Implements a pity/mercy system that guarantees a drop after N failures. Tracks state per-table (or per shared group) using integer keys. Implements IResettable so callers can reset all counters via the interface.

PityTracker

Default in-memory implementation of IPityTracker. Stores failure counts in a dictionary.

SystemRandom

Default implementation using System.Random.

UniqueDropStrategy<T>

Tracks items that have already dropped and prevents duplicates. Useful for one-time drops (e.g., unique quest items, first-time boss kills).

UnityRandom

Implementation using Unity's random. Not seedable per-instance.

WeightedRandomStrategy<T>

Generates loot results using weighted random selection. This is typically the first strategy in a pipeline.

Factory interface for converting loot definitions into actual item instances. Users implement this to define how their items are created from loot results.

A strategy that selects items from the table and adds them as results. These strategies are typically skipped during the "dry-run" preview.

Observes completed loot rolls. Register via AddObserver(ILootObserver<T>). Called after all strategies have finished processing.

ILootResultModifierStrategy<T>

A strategy that modifies loot results after they have been generated.

A single step in the loot pipeline. Strategies can generate, filter, or modify loot results.

ILootTableModifierStrategy<T>

A strategy that modifies the loot table (weights, entries) before any items are rolled. These strategies are executed during the "dry-run" preview.

ILootTable<T>

Represents a collection of loot entries that can be rolled against.

IPityTracker

Tracks failure/success counts for a pity system. Can be used standalone or within the loot pipeline.

IRandom

Abstraction for random number generation, allowing for deterministic/seeded randomness.

IResettable

Implemented by stateful strategies that support full state reset. Use pipeline.Strategies.OfType<IResettable>() to reset all stateful strategies from Unity lifecycle hooks (OnDestroy, scene reload, etc.) without importing concrete strategy types.
